# 运行资源 UI 规范

适用范围：右侧健康自检抽屉里的“运行资源”模块，以及缺失资源时弹出的阻断提示。

## 信息结构

- 模块标题固定为“运行资源”，只放在 `drawer-card-head`。
- 每个资源项使用独立容器 `runtime-resource-item`，不要复用健康项 `health-item`。
- 单项内容顺序固定为：资源名称、状态、说明、操作按钮。
- 如果资源支持本地文件安装，说明和操作按钮之间必须显示“本地文件”小字提示，内容为 manifest 下载 URL 对应的文件名。
- 状态只表达当前可用性，例如“可用”“missing”“error”，不要用状态标签替代说明文案。

## 间距与尺寸

- 资源列表间距：`14px`。
- 资源项内边距：深色模式 `14px`，浅色窄抽屉覆盖为 `12px`。
- 资源项内部纵向间距：`12px`。
- 标题行最小高度：`42px`，保证状态胶囊垂直居中。
- 状态胶囊最小尺寸：`60px × 34px`。
- 操作按钮间距：`8px`，按钮最小高度：`40px`。

## 对齐与自适应

- 标题行使用两列网格：左侧 `minmax(0, 1fr)`，右侧状态自适应内容宽度。
- 资源名称允许换行，使用 `overflow-wrap: anywhere` 防止长名称撑破抽屉。
- 说明文字允许换行，字号 `12px`，行高 `1.65` 左右。
- 文件名提示使用 `12px` 正文、`11px` 标签、`10px 12px` 内边距，长文件名必须在卡片内换行。
- 操作区在右侧抽屉内始终单列、全宽、等高，不在窄宽度里横向挤压。
- 不要在运行资源项里嵌套第二层卡片，也不要让按钮组混用通用 `button-row` 的横向布局。

## 状态与交互

- “可用”状态使用项目主强调色 `--syn-capture`，其余状态使用低强调表面色。
- “下载”在资源已可用、缺少下载地址、缺少 SHA256 或当前忙碌时禁用。
- “选择本地文件”和“刷新”只在当前资源忙碌时禁用。
- 本地文件选择器必须是工作台级隐藏控件，弹窗和右侧抽屉共用同一个上传入口，避免抽屉内按钮找不到 input。
